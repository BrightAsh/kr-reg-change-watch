# 한국 규제 변경 모니터

법령, 고시·공고, 지침·행정규칙, 입법예고, 행정예고, 정책자료, 뉴스 보조 자료를 자동으로 수집해 날짜별로 보여주는 규제 변경 모니터링 앱입니다. 사용자는 캘린더에서 날짜를 선택하고, 기관·출처·문서유형·변경유형 필터로 필요한 자료를 빠르게 확인할 수 있습니다.

서비스 페이지: https://brightash.github.io/kr-reg-change-watch/

## 앱 소개

이 앱은 일하는 사람이 “오늘 어떤 규제성 변경이 있었는지”를 한 화면에서 확인하기 위한 업무용 모니터링 도구입니다.

주요 기능은 다음과 같습니다.

| 기능 | 설명 |
| --- | --- |
| 날짜별 조회 | 캘린더에서 날짜를 선택하면 해당 일자에 수집된 자료만 보여줍니다. |
| 자동 분류 | 수집 자료를 법령, 고시/공고, 지침/규칙, 뉴스/발언으로 분류합니다. |
| 필터와 검색 | 기관, 출처, 문서유형, 변경유형, 검색어로 자료를 좁혀 봅니다. |
| 상세 화면 | 각 항목의 원문 링크, 기본 정보, 수집 본문, 첨부 링크를 확인합니다. |
| AI 브리핑 | 사용자가 AI 모델과 키를 설정하면 현재 목록 전체를 업무용 브리핑으로 요약합니다. |
| AI 요약 | 상세 화면에서 수집 본문을 기준으로 개별 항목을 요약합니다. |

## 앱 환경과 운영 방식

| 항목 | 내용 |
| --- | --- |
| 웹사이트 배포 | GitHub Pages로 정적 웹사이트를 배포합니다. |
| 자동 수집 | GitHub Actions가 매일 한국시간 0시 기준으로 자동 실행됩니다. |
| 수집 방식 | 공식 Open API, 공공데이터 API, RSS, 정부 부처 게시판 HTML 수집을 함께 사용합니다. |
| 데이터 관리 | 수집 결과는 날짜별 JSON 데이터로 정리되어 화면에서 바로 조회됩니다. |
| 캐시 방식 | 이미 해당 날짜 자료가 있으면 기존 데이터를 우선 보여주고, 강제 수집 시 새로 수집합니다. |
| 사용자 화면 | 별도 로그인 없이 GitHub Pages 웹사이트에서 조회할 수 있습니다. |

## 수집 출처와 방식

| 구분 | 수집 항목 | 수집 방법 | 필요한 키/설정 | 현재 상태 |
| --- | --- | --- | --- | --- |
| 국가법령정보센터 | 법령 변경이력, 조문 개정 이력, 개정문/제개정이유, 행정규칙, 행정규칙 신구법 비교 | 대상 날짜로 Open API를 조회하고, 여러 행으로 내려오는 자료는 법령ID/행정규칙ID 단위로 묶어 저장 | `LAW_OPEN_API_OC` | 정상 |
| 국민참여입법센터 | 법제처·행안부·기재부 입법예고, 입법예고 수정분, 행정예고 | 입법예고 목록, 입법예고 목록(수정일 기준), 행정예고 목록 API를 대상 날짜로 조회하고 상세 API로 본문과 첨부를 보강 | `LAWMAKING_OC` | 키 승인 요청 중 |
| 대한민국 전자관보 | 관보 고시, 공고, 법령 공포성 자료 | 관보 목록 API URL에 대상 날짜와 서비스키를 붙여 조회 | `GWANBO_LIST_URL`, `DATA_GO_KR_SERVICE_KEY` | 정상 |
| 행정안전부 게시판 | 훈령·예규·고시, 입법·행정예고 | 공식 게시판 HTML 목록과 상세 페이지를 읽고 대상 날짜 글만 저장 | 별도 키 없음 | 정상 |
| 기획재정부 게시판 | 훈령, 예규, 고시, 공고, 지침, 입법예고, 행정예고 | 공식 게시판 HTML 목록과 상세 페이지를 읽고 대상 날짜 글만 저장 | 별도 키 없음 | 정상 |
| 정책브리핑 RSS | 보도자료, 정책뉴스, 부처 RSS, 대통령·연설·국무회의·브리핑 RSS | RSS 발행일이 대상 날짜이고 규제 관련 키워드가 있는 항목만 저장 | 별도 키 없음 | 정상 |
| 네이버 뉴스 검색 | 규제·법령·고시·지침 관련 뉴스 | 기본 검색어를 뉴스 API로 조회하고 대상 날짜 기사만 저장 | `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET` | 정상 |

뉴스, 정책브리핑, 대통령·장관 발언성 자료는 공식 변경 확정 자료가 아니라 참고 자료입니다. 화면에서는 확인이 필요한 보조 자료로 취급합니다.

수집기는 외부 출처에서 가져온 항목을 그대로 누적하지 않고, 아래 방식으로 정리합니다.

| 처리 단계 | 내용 |
| --- | --- |
| 날짜 필터링 | 대상 날짜에 수집되었거나 대상 날짜에 공표된 자료만 해당 일자 자료로 저장합니다. |
| 중복 정리 | 법령ID, 행정규칙ID, 입법예고 일련번호, 원문 URL, 안정 ID를 기준으로 중복 항목을 병합합니다. |
| 본문 확보 | 가능한 경우 상세 페이지, 원문 링크, 첨부 링크, API 원문 데이터를 함께 저장합니다. |
| 분류 | 법령, 고시/공고, 지침/규칙, 뉴스/발언으로 자동 분류합니다. |
| 신뢰도 표시 | 공식 API/게시판은 공식 자료로, 뉴스·정책자료는 확인 필요 자료로 표시합니다. |

## AI 모델 사용

AI 기능은 사용자가 모델 제공자와 API 키를 직접 입력했을 때만 동작합니다. 한 번 키를 저장하면 같은 브라우저에서 다음부터는 AI 브리핑 또는 AI 요약 버튼을 누르는 즉시 실행됩니다.

선택 가능한 모델 제공자는 Gemini, Groq, OpenRouter, OpenAI입니다.

| 기능 | 입력 자료 | 결과 | 비용 |
| --- | --- | --- | --- |
| AI 브리핑 | 현재 화면에 표시된 여러 항목의 제목, 기관, 날짜, 본문 일부 | 선택 날짜 또는 필터 결과 전체를 업무용 브리핑으로 요약 | 선택한 모델 제공자의 정책에 따라 무료 한도 또는 비용 발생 |
| AI 요약 | 상세 화면의 수집 본문 | 개별 항목의 핵심 내용, 적용 대상, 확인할 업무 영향을 요약 | 선택한 모델 제공자의 정책에 따라 무료 한도 또는 비용 발생 |

AI 요약은 현재 앱이 이미 수집한 본문을 기준으로 동작합니다. 원문 URL을 AI가 직접 웹 검색해서 읽는 방식은 기본 동작이 아닙니다. 원문 본문을 더 많이 확보할수록 AI 요약 품질도 좋아집니다.

API 키는 사용자의 브라우저에 저장됩니다. 공용 PC에서는 사용 후 앱 화면에서 키를 삭제하는 것이 좋습니다.

## chrisryugj/korean-law-mcp 검토 내용

[chrisryugj/korean-law-mcp](https://github.com/chrisryugj/korean-law-mcp)는 국가법령정보센터 등 한국 법령 API를 MCP 서버 형태로 사용할 수 있게 만든 도구입니다. MCP는 ChatGPT나 Claude 같은 AI 도구가 외부 기능을 호출할 수 있도록 연결하는 서버 방식입니다.

이 앱에서는 해당 MCP 서버를 그대로 실행하지 않았습니다. 이유는 다음과 같습니다.

| 항목 | 설명 |
| --- | --- |
| GitHub Pages 제약 | GitHub Pages는 정적 웹사이트 배포 환경이라 MCP 서버나 별도 Node 서버를 상시 실행할 수 없습니다. |
| 비밀키 관리 | 브라우저와 정적 페이지 안에 법령 API 키를 직접 넣으면 노출 위험이 있습니다. |
| 자동 수집 구조 | 이 앱은 GitHub Actions에서 매일 수집한 결과를 JSON으로 저장하고, 웹사이트는 그 결과를 읽는 구조입니다. MCP 서버 상시 호출 구조와 다릅니다. |

다만 해당 레포를 검토하면서 적용 가능한 방향은 반영했습니다.

| 반영 방향 | 내용 |
| --- | --- |
| API 중심 수집 | 브라우저 화면을 무리하게 긁기보다 공식 API 응답을 우선 사용하도록 정리했습니다. |
| 법령 본문 보강 | 법령 변경이력에서 가능한 경우 개정문, 제개정이유, 법령 본문을 추가로 확보하도록 보강했습니다. |
| 키 이름 유연화 | 국가법령정보센터 키를 여러 환경변수 이름으로 인식할 수 있게 했습니다. |
| 상세 데이터 우선 | 목록만 보여주는 것이 아니라 상세 API와 원문 데이터를 함께 저장하는 방향으로 정리했습니다. |

요약하면, `korean-law-mcp`를 그대로 붙인 것은 아니지만, “공식 법령 API를 구조화해서 AI와 앱이 쓰기 좋은 데이터로 바꾼다”는 방향은 현재 수집 로직에 반영했습니다.
